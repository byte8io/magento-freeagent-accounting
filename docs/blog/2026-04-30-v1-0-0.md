The first public release of **Byte8 FreeAgent Accounting** is out. SaaS connector for **Magento 2 → FreeAgent** — invoices, credit notes, customers, products, and payments syncing within minutes, with full sync-status visibility from the Magento admin and side-by-side coexistence with Sage Accounting on the same Magento install.

<!-- truncate -->

## What's in v1.0.0

### Connect flow

- **Pairing-code Connect** — generate a 30-min code in Magento admin, paste into `ledger.byte8.io`, the chassis handshakes back. No OAuth client secret on disk, no callback URL wrangling.
- **FreeAgent OAuth** — fully chassis-side. Tokens encrypted at rest (AES-GCM), refreshed transparently, never seen in PHP. Single global API endpoint (`api.freeagent.com/v2`); sandbox available on request.
- **Per-provider routing** — outbox carries `provider` column; webhook URL carries `?provider=freeagent` query. Sage + FreeAgent can pair on the same tenant + the same Magento install without colliding.

### Outbound sync (Magento → FreeAgent)

- **Invoices** (`invoice.created`) — Open AR with full line items, addresses, currency, per-line discounts, dedicated shipping line. Always carries `payment_terms_in_days` (FreeAgent quirk #1).
- **Payments** (`invoice.paid`) — `bank_transaction_explanation` against the matching FreeAgent invoice URL, routed per the per-tenant payment-method map. Invoice transitions Open → Paid in FreeAgent.
- **Credit notes** (`creditmemo.created`) — refunds routed to the same contact as the original invoice, with shipping refund line and original-invoice date for accountant reporting.
- **Customers** (`customer.upserted`) — Magento customer → FreeAgent contact. Lookup-by-email self-heal on duplicate POST 422.
- **Products** (`product.upserted`, opt-in) — simple → FreeAgent `Products`, virtual / downloadable → `Services` by default. Per-Magento-type item-type override available. Composite types skipped intentionally.

### Coexistence with Sage Accounting

- Both connectors share the `byte8/module-client` chassis (outbox, JWT auth, sync-state mirror). Outbox `provider` column + per-provider grid join aliases (`byte8_sync_fa` table alias, `_freeagent` column suffix) keep the two from trampling each other.
- Sales → Invoices grid renders **two columns** when both connectors are installed — "Sage Status" + "FreeAgent Status" — each with its own chip and tooltip. Same on credit-memo grid + invoice / credit-memo detail pages.
- Independent pairing + disconnect surfaces; one provider's outage doesn't block the other.

### Magento admin visibility (PR7)

- **FreeAgent Status column** on Sales → Invoices and Sales → Credit Memos grids. Sortable + filterable. Chips: ✓ Synced / ⏳ Pending / ⏸ Skipped / ✗ Failed / —. Hover for FreeAgent invoice URL, skip-reason, or error code.
- **FreeAgent Accounting info block** on every Invoice + Credit Memo detail page — chip, FreeAgent URL, last sync timestamp, skip / error context.
- **Write-through pending mirror** — chip appears the moment the merchant clicks Submit Invoice, not 60 s later when the cron drain catches up.
- **Dead-letter banner** on the config page surfaces failed deliveries + links to the `byte8:client:outbox:inspect` triage CLI (provider-filterable via `--provider=freeagent`).

### Operator surfaces

- **Console commands** for outbox triage: `byte8:client:outbox:{inspect,requeue,cleanup}`. The previous `byte8:sage:outbox:*` namespace is renamed — outbox triage is provider-agnostic and lives on the shared chassis CLI.
- **Settings dashboard** at `ledger.byte8.io/dashboard/bindings/{id}/settings` with five cards: Sync Behaviour, FreeAgent Defaults, Payment-method Map, Item-type Map, Commercial Knobs. Server-side validation against the live FreeAgent reference cache; red-underline field errors before save.
- **Sync history** dashboard at `ledger.byte8.io/dashboard/sync` with full per-event audit + per-row retry button. Provider filter chip lets you scope to FreeAgent rows only.
- **Disconnect dialog** — when disconnecting from the chassis dashboard, a structured "What gets removed" / "What stays" breakdown surfaces before the operator commits.

### Catalogued FreeAgent quirks

Two non-obvious FreeAgent v2 behaviours captured + worked around:

- **§1 — `payment_terms_in_days` required on every POST.** Even on net-cash invoices, even on £0 lines. Chassis always sends a value (default 30, configurable per binding).
- **§2 — Numeric fields stringified on responses.** `exchange_rate: "1.0"`, `total_value: "100.00"`, etc. Chassis decodes create responses through minimal envelopes that only read `url` to dodge the asymmetric encoding.

Full catalogue: `apps/ledger/__docs/FREEAGENT_API_QUIRKS.md`. New entries land any time the worker logs a 4xx that isn't an obvious operator typo.

### Distribution

- `byte8/magento-freeagent-accounting` on Cargoman (Byte8 private Composer registry) for the closed-beta cohort. Public Packagist + Magento Marketplace listing follow.

## Intentionally NOT in v1

- **No FreeAgent Status chip on Sales → Orders or Customers grids.** Orders aren't synced directly (we sync invoices, 0..N per order); customer mirror is single-row but the chassis doesn't publish contact-resolution status as a grid signal yet. Order-rollup info block + customer-block surfacing are planned v1.1+.
- **No `provider_reference`** (e.g. FreeAgent's human-readable invoice number) populated on the chip yet. Column exists in the schema; chassis sends `null` today. v1.1+ — needs a getter on the provider trait so `reference` rides back with the FreeAgent create response.
- **No manual "Resync now" button on Magento detail pages.** Operators use the chassis dashboard's per-row retry; no need to duplicate the surface yet.
- **No standalone `payment.captured`** for offline payments. Magento has no API to attach an offline payment to an existing invoice — accountants reconcile manually in FreeAgent's bank feed when the cheque / bank transfer lands.
- **No FreeAgent → Magento writeback.** Enterprise on request — needs FreeAgent webhook surface + Magento write endpoints + conflict-resolution policy. Build only on a custom contract.
- **No stock-level sync.** FreeAgent's product catalog isn't designed to track inventory; merchants who need stock sync use Sage Accounting instead.
- **No projects / timeslips.** Out-of-scope for an e-commerce connector.
